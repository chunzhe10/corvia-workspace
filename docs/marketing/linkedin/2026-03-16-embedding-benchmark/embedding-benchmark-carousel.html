The Nuance
What the Numbers
Don't Tell You
A benchmark is only as honest as its caveats. Here are the things I'd want to know before drawing conclusions from these numbers.
⚠
CPU short-text anomaly is misleading
527ms looks terrible, but it's ORT's batch startup cost spread over only 5 items. With 10+ items, CPU drops to 56ms/embed — nearly matching GPU backends.
📋
Pipeline overhead dominates
~45-50ms of every request is CLI + HTTP + gRPC + HNSW search. Pure inference time is only ~5-10ms on GPU. The backend matters less than you'd think.
🚀
NVIDIA wins on different workloads
Bigger models, longer sequences, larger batch sizes — that's where 6 GB VRAM and 3840 CUDA cores pull ahead. This test used a 137M param model with small batches.
🛠
This is one machine
A laptop i7 + RTX 3060. Your hardware will differ. The pattern (shared memory wins for small models) likely generalizes, but the specific numbers won't.

The takeaway: these numbers are directionally useful, but your mileage will vary. Always benchmark on your own hardware.

What It Took
Behind

One command to switch backends. Getting there was a different story.
0
PACKAGES
CPU
ORT default. Just works.
13
PACKAGES
Intel iGPU (OpenVINO)
5 Intel compute-runtime packages • 3 libtbb .debs cross-pulled from Ubuntu Noble (not in Debian Trixie) • 3 OpenVINO libs from Intel apt repo • manual symlinks because APT installs to /usr/lib/ but ORT checks /usr/lib/x86_64-linux-gnu/ • /dev/dri by-path symlinks for GPU discovery
5+
PACKAGES
NVIDIA dGPU (CUDA)
NVIDIA's apt repo uses SHA1-signed GPG keys — Debian Trixie rejects them. Had to install cublas, curand, cufft, cudart, cuDNN via pip instead • pip buries .so files deep in site-packages — manual symlink tree into /usr/lib/ • CDI spec auto-regen on driver version change
# After all that, the user just runs:
$ corvia use openvino  Reload complete.
$ corvia use cuda      Reload complete.
$ corvia use cpu       Reload complete.

The whole point: absorb the complexity so switching backends is one command, no restart, no config files.

Key Takeaways
What I Learned
From These Numbers
📈
In RAG, retrieval matters more than the model.
89% of pipeline time was search + graph — not inference. Optimize the retrieval layer first.
⚡
Shared memory beats raw compute for small models.
If data transfer costs more than the computation, zero-copy wins.
🛠
Don't discard your iGPU.
It's free compute sitting idle. For embeddings, it can match or beat a discrete GPU.
🔌
Make backend switching painless.
Different hardware, different optimal backends. Let users switch at runtime.
