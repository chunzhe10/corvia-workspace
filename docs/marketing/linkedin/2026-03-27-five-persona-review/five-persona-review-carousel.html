The Problem
Solo Dev.
No Code Review.
I'm building a Rust project solo. No team. No PR reviews.
The bugs I was shipping had a pattern.
Bugs That Shipped
❌
Config typo silently swallowed
A wildcard match caught all strings. Misspell a retriever name and it just... works, doing nothing.
❌
CLI returned success on failure
Server unreachable? Exit code 0. Looked like success in every CI pipeline.
❌
Race condition in log rotation
Size check was outside the mutex. A TOCTOU race that passed every test.
❌
DoS vector in dashboard
No file size limit on log reads. One large file could OOM the server.

These aren't edge cases. They're the kind of bugs that pass every test and break in production. The kind a teammate would catch in a PR review.

Key Takeaways
What I Learned
🎯
Roles change everything
A generic "review this" catches surface issues. A PM persona catches incomplete work. A Security persona catches leaked secrets. The role defines what gets found.
📈
Independent context windows matter
One reviewer with five roles collapses to one perspective. Five separate subagents find different things because they think independently.
📄
Review designs, not just code
The highest-leverage reviews happen before implementation. A flaw caught in a spec costs minutes. In code, it costs hours.
⚡
Speed without review is shipping bugs faster
AI makes writing code fast. The real leverage is simulating the team dynamics that catch mistakes before they ship.
