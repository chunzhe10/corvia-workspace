My AI agent builds the product. Tests it. Makes the commits. But until recently, it couldn't tell me why a search took 120ms instead of 40ms. So I wired OpenTelemetry traces into every kernel operation. 30 span constants, defined before the code existed. Every RAG query, embedding, and store operation gets a parent-child trace tree with p50/p95/p99 latency. Slide 4 has the raw data from a live session. The first thing the agent found: 60-85% of RAG pipeline time was in vector retrieval, not embedding. I had been assuming the model was the bottleneck. The traces said otherwise. But observability alone is a dashboard with no buttons. I wanted to operate the system, not just watch it. So I added 10 control plane tools to the same MCP server. Three safety tiers built on MCP's existing primitives. Read freely. Mutate with confirmation. Destructive ops need a dry-run first. Slide 6 has the full breakdown. What this leads to: observe, hypothesize, change a config value via hot-reload, measure. All in one conversation. The product becomes a lab bench. The agent is both the experimenter and the instrument (slide 7). I'm still early. The tools are shipped but the workflows around them are still forming. MCP standardized how agents talk to tools. There's no standard yet for how agents operate the systems they talk to. Someone needs to make that plug-and-play. What's the most interesting thing you've let an AI agent change in a running system?
